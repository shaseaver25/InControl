## The big picture

```
[Nick fills brief] → [Generator calls Claude API] → [Nick reviews 8 step cards]
       → [Nick adds video] → [Nick approves + publishes]
              ↓
     [lessons + lesson_steps rows in Supabase]
              ↓
     [YOUR LEARNER APP reads and renders]
```

When a lesson is published, two things exist in Supabase:
- One row in `lessons` (metadata)
- Eight rows in `lesson_steps` (one per step, ordered 1–8)

Your app reads those rows and renders them as a guided, interactive lesson experience.

## Your primary query

```typescript
// Get all steps for a lesson, in order
const { data: steps } = await supabase
  .from('lesson_steps')
  .select('*')
  .eq('lesson_id', lessonId)
  .eq('status', 'published') // only show published lessons
  .order('step_number');

// Get published lessons (for a lesson list/home screen)
const { data: lessons } = await supabase
  .from('lessons')
  .select('id, title, domain, month, status')
  .eq('status', 'published')
  .order('created_at');
```

## The 8 lesson steps — always in this order

Every lesson has exactly 8 steps. The `step_type` tells you which component to render. The `content` field is a JSON object with step-type-specific fields.

| step_number | step_type | What it is |
|---|---|---|
| 1 | `opener` | Greeting + mood check |
| 2 | `goal_connect` | Links today's lesson to learner's personal goal |
| 3 | `teach` | The concept, key points, and video |
| 4 | `guided_practice` | We do together — hint shown before any tap |
| 5 | `independent` | Learner tries alone — immediate feedback |
| 6 | `choice_point` | Learner picks how to apply the skill today |
| 7 | `commitment` | One tiny behavior anchored to a daily routine |
| 8 | `celebration` | Specific praise, progress, next lesson preview |

## Content schemas — what's inside `content` for each step type

These are the exact field names. All are generated by Claude and reviewed by Nick before publish.

### `opener`
```json
{
  "greeting": "Hi, [Name]! Today we learn about exercises that make your body stronger.",
  "mood_check": {
    "scale": "5-emoji",
    "options": ["😔", "😐", "🙂", "😄", "🤩"]
  }
}
```
- Replace `[Name]` with the learner's name at render time
- Render all 5 emoji as large tappable buttons
- Auto-advance to step 2 when learner taps one

### `goal_connect`
```json
{
  "connection_sentence": "Learning about resistance training helps you feel stronger and have more energy for everyday activities.",
  "progress_note": "You are building knowledge about exercises that work for your body. Great start!"
}
```
- Show learner's personal goal (from their profile) prominently
- Show connection_sentence linking today's lesson to that goal
- Show progress_note as an encouraging sub-note

### `teach`
```json
{
  "headline": "What is resistance training?",
  "key_points": [
    { "emoji": "💪", "text": "Builds and maintains muscle strength." },
    { "emoji": "⚖️", "text": "Improves balance and coordination." },
    { "emoji": "🏡", "text": "Helps with daily tasks like lifting and walking." }
  ],
  "video_plan": {
    "title": "Introduction to Resistance Training",
    "duration": "60-90 seconds",
    "setting": "Living room or home gym area",
    "props": ["resistance band", "chair"],
    "script": "Hi [Name]! I'm Coach Maya. Today we're going to talk about resistance training...",
    "visuals": "Open on Coach Maya standing with a resistance band. Cut to demonstration...",
    "key_shots": [
      "Close-up of resistance band being held",
      "Coach demonstrating a basic band pull",
      "Wide shot of Coach smiling at camera"
    ]
  },
  "video_url": "https://www.youtube.com/watch?v=..." 
}
```
- `video_url` may be null if Nick hasn't added the video yet — show a "coming soon" placeholder
- If `video_url` contains "youtube.com" or "youtu.be", embed as YouTube iframe
- If `video_url` is a direct file URL, use `<video>` element
- Captions ON by default
- Audio narration ON by default
- `video_plan` is for Nick's production team — do not show to learners

### `guided_practice`
```json
{
  "question": "Which drink helps your muscles the most?",
  "choices": [
    { "emoji": "🥤", "label": "Water" },
    { "emoji": "🧃", "label": "Juice" },
    { "emoji": "🍬", "label": "Candy" },
    { "emoji": "☕", "label": "Coffee" }
  ],
  "correct_answer_label": "Water",
  "hint": "Think about what Coach Maya showed in the video — the drink that helps plants grow.",
  "success_feedback": "You remembered that water helps muscles work — great thinking!",
  "guidance_feedback": "Water is the one — tap the water glass!"
}
```
**Critical interaction rules:**
- Show `hint` BEFORE the learner taps anything — this is errorless learning, not a reveal
- If learner taps the wrong choice: show `guidance_feedback` (never say "wrong") and highlight the correct answer
- If learner taps the correct choice: show `success_feedback` immediately (under 2 seconds)
- Feedback must be both visual (color change, checkmark) AND auditory (TTS narration)
- Auto-advance after correct answer + feedback display

### `independent`
```json
{
  "question": "When is the best time to drink your first glass of water?",
  "choices": [
    { "emoji": "🌅", "label": "Morning" },
    { "emoji": "🌙", "label": "Bedtime" },
    { "emoji": "🎬", "label": "Movies" }
  ],
  "correct_answer_label": "Morning",
  "success_feedback": "You picked morning — that's when your body needs water most after sleeping!",
  "guidance_feedback": "Morning is the best time — tap the sunrise!"
}
```
- Same interaction rules as `guided_practice` EXCEPT: no hint shown before tapping
- One wrong tap allowed before showing guidance_feedback + highlighting correct answer

### `choice_point`
```json
{
  "prompt": "How will you remember to drink water today?",
  "choices": [
    { "emoji": "🥤", "label": "Water bottle by bed", "description": "See it right when you wake up" },
    { "emoji": "⏰", "label": "Phone reminder", "description": "Gets a notification at 7am" },
    { "emoji": "📝", "label": "Sticky note on mirror", "description": "See it while getting ready" },
    { "emoji": "🍳", "label": "While breakfast cooks", "description": "Already part of your morning" }
  ]
}
```
- All choices are valid — no correct answer
- Learner just picks one and advances
- Large cards with emoji + bold label + description below
- Selection is recorded (optional — for future analytics)

### `commitment`
```json
{
  "anchor": "Making breakfast",
  "action": "Fill my water bottle and put it on the counter.",
  "anchor_emoji": "🍳",
  "action_emoji": "🥤",
  "reminder_time": "07:30"
}
```
- Render as: "After I [anchor_emoji] **[anchor]**, I will [action_emoji] **[action]**"
- Show as a visually distinct card the learner "accepts"
- Optionally schedule a push notification at `reminder_time`
- Save to `commitments` table when learner confirms

### `celebration`
```json
{
  "praise": "You learned what resistance training is and named one way it helps your body!",
  "progress_note": "Lesson 1 of your Resistance Training goal is done!",
  "next_lesson_preview": "Why Resistance Bands Work for You"
}
```
- Play a celebration animation/sound first
- Show `praise` (specific to this lesson — not generic)
- Show progress bar or visual progress indicator
- Show `next_lesson_preview` so learner knows what's coming

## Accessibility requirements — non-negotiable

These are not UI polish. For this population (IDD/ASD), getting these wrong breaks the product.

**1. Touch targets: minimum 56px on all interactive elements.**
No exceptions. Spacing between targets: minimum 8px.

**2. One thing at a time.**
One question per screen. One concept per step. Max 4 choices. Never stack two interactions.

**3. Audio narration on by default.**
Every text element needs a TTS equivalent. Users should be able to hear every screen without reading.

**4. Captions on by default.**
All video has captions. Never autoplay with sound.

**5. Immediate feedback — under 2 seconds.**
When a learner taps a choice, the response (visual + audio) must arrive within 2 seconds. Delayed feedback breaks the learning loop.

**6. Never show failure.**
No red X. No "wrong." No "try again." Guidance only. The word "wrong" must never appear.

**7. Visual schedule always visible.**
Show all 8 step icons at the top of every screen (done ✓ / active → / upcoming). Never hide it. This reduces anxiety by making the session predictable.

**8. Consistent coach identity.**
Same avatar, same name, same greeting style on every step. Predictability is the engagement mechanism.

**9. Streak language: additive only.**
"You've done this 12 times this month." Never "You broke your 12-day streak."

**10. Structured choice only.**
2–4 options with emoji + label. No open text input as the default. If you need text input anywhere, it must be optional.

## The learning graph (optional — for future personalization)

The `concepts` table (133 rows) and `concept_edges` table (177 rows) contain the full prerequisite graph for all lesson content. You don't need this for the MVP, but if you want to show learners a progress map or unlock lessons sequentially, the data is there.

```sql
-- Get all concepts in a domain
SELECT id, label, role, month FROM concepts WHERE category_name = 'Resistance Training';

-- Get what a concept depends on (prerequisites)
SELECT source FROM concept_edges WHERE target = 'squat';

-- Get what a concept unlocks (what comes after)
SELECT target FROM concept_edges WHERE source = 'what_resistance';
```

## What to build first

Build in this order:

1. **Lesson player shell** — the chrome: visual schedule bar (8 step icons), coach avatar/name chip, audio button, navigation (next/back). Static for now.
2. **Step renderer** — one component per step_type, rendering from the `content` JSON. Start with `opener` and `goal_connect` (no interaction logic needed).
3. **Choice interaction** — `guided_practice` and `independent` steps with the errorless hint and feedback flow. This is the hardest to get right and the most important.
4. **Commitment step** — render + optional push notification + write to `commitments` table.
5. **Celebration step** — animation + progress bar.
6. **Lesson list / home screen** — query `lessons` table, show published lessons.
7. **Video embed** — YouTube iframe or `<video>` for the teach step.
